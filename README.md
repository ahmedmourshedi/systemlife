# نظام حياتي الشخصي Mega

مشروع عربي كامل يعمل كـ **موقع إلكتروني + تطبيق PWA على الجوال + داشبورد على اللابتوب** لإدارة الحياة الشخصية بشكل احترافي.

التقنيات المستخدمة:

- **Next.js / React / TypeScript** للواجهة والصفحات.
- **Supabase** للمصادقة وقاعدة البيانات و Row Level Security.
- **Tailwind CSS** للواجهة العربية RTL.
- **Vercel** للنشر.
- **PWA** لتثبيت الموقع على شاشة الجوال وفتحه كتطبيق.

## تحديث التصميم والتجاوب

تمت إضافة نسخة تصميم محسنة جدًا وفق مبادئ UI/UX وتشمل:

- هوية فاتحة مريحة للعين بدل الخلفية الداكنة.
- خط عربي واضح يعتمد على IBM Plex Sans Arabic مع بدائل عربية مناسبة.
- أيقونات SVG احترافية داخلية بدل الاعتماد على الإيموجي كواجهة بصرية.
- تباين أفضل للقراءة وأحجام خط أكثر هدوءًا على الجوال واللابتوب.
- Sidebar مقسم إلى مجموعات للابتوب.
- Bottom Navigation للجوال.
- Drawer كامل للجوال يحتوي كل الوحدات.
- بطاقات ونماذج وجداول أكثر وضوحًا.
- دعم أفضل للـ RTL و Safe Area على الجوال.
- صفحة تسجيل دخول وإنشاء حساب بتصميم احترافي.
- صفحة إدارة مستخدمين متجاوبة: بطاقات على الجوال وجدول على اللابتوب.

راجع التفاصيل في:

```text
docs/design-system.md
RESPONSIVE_DESIGN_CHANGELOG.md
UI_UX_FINAL_CHANGELOG.md
DESIGN_REVIEW_CHECKLIST.md
```

## متغيرات البيئة

ضع القيم في `.env.local`:

```env
NEXT_PUBLIC_SUPABASE_URL=https://YOUR_PROJECT.supabase.co
NEXT_PUBLIC_SUPABASE_PUBLISHABLE_KEY=YOUR_SUPABASE_PUBLISHABLE_KEY
NEXT_PUBLIC_SITE_URL=http://localhost:3000
```

عند النشر على Vercel غيّر `NEXT_PUBLIC_SITE_URL` إلى رابط المشروع النهائي.

## إعداد قاعدة البيانات

1. افتح Supabase.
2. ادخل إلى SQL Editor.
3. شغل الملف:

```text
database/schema.sql
```

هذا الملف يحتوي الجداول الأساسية + توسعة Mega.

إذا كنت سبق وشغلت النسخة القديمة، يمكنك تشغيل ملف التوسعة فقط:

```text
database/mega-extension.sql
```

---

## جعل أول مستخدم أدمن

بعد إنشاء أول حساب من صفحة التسجيل، افتح جدول `profiles` في Supabase وخذ معرف المستخدم، ثم شغل:

```text
database/first-admin.sql
```

واستبدل `USER_ID` بمعرف المستخدم الحقيقي.

---

## بيانات تجريبية واقعية

لإظهار التطبيق كأنه يعمل على بيانات حقيقية، شغل ملف البيانات الواقعية بعد إنشاء أول حساب:

```text
database/realistic-demo-seed.sql
```

هذا الملف يضيف بيانات مترابطة للمال، الدخل، الميزانيات، الاشتراكات، الصحة، الوجبات، الوزن، المياه، التمارين، الكورسات، الكتب، الأهداف، المهام، العادات، الاستثمار، المشاريع، العلاقات، السفر، الأتمتة والتنبيهات.

ويوجد مجلد بيانات أداء كبير:

```text
sample-data/performance
```

الغرض من ملفات الأداء هو اختبار الاستيراد، البحث، التصدير، وثقل الداشبورد. لا ترفعها إلى Supabase كلها في البداية إلا إذا كنت تريد تجربة ضغط حقيقية.

## ملاحظات مهمة

- لا تضع `node_modules` داخل ملف zip ولا ترفعه إلى GitHub.
- لا تحفظ كلمات مرور البنوك أو المحافظ داخل النظام.
- التنبيهات الحالية مسجلة داخل قاعدة البيانات، وليست Push Notifications حقيقية بعد.
- رفع الملفات مجهز على مستوى قاعدة البيانات و Storage Policies، ويمكن تطوير واجهة الرفع لاحقًا.
- النظام مصمم كنسخة عربية فقط، وواجهة RTL بالكامل.

## فلسفة التصميم

النظام ليس دفتر تسجيل فقط. الفكرة أن يكون **Operating System لحياتك الشخصية**:

- تلتقط بسرعة.
- تصنف يوميًا.
- تراجع أسبوعيًا.
- تحلل شهريًا.
- تربط المال والصحة والتعلم والأهداف ببعض.
- تستخدم الجوال للإدخال السريع واللابتوب للتحليل والإدارة.
