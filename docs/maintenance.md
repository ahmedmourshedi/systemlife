# الصيانة والتشغيل

## أسبوعيًا

- راجع أخطاء Vercel.
- راجع جداول جودة البيانات.
- راجع حجم Storage.
- جرب تسجيل دخول وخروج.

## شهريًا

- صدّر نسخة JSON.
- راجع جداول غير مستخدمة.
- راجع صلاحيات الأدمن.
- راجع أداء التقارير.

## قبل أي تعديل كبير

1. صدّر قاعدة البيانات.
2. انسخ `schema.sql`.
3. جرّب التعديل على مشروع Supabase تجريبي.
4. اختبر RLS.
5. انشر على Vercel Preview أولًا.
