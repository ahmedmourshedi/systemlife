# دليل الاستخدام الشخصي لحساب واحد

هذا المشروع مضبوط الآن ليعمل كنظام شخصي لحساب واحد فقط.

## الحساب الأساسي

```text
admin@lifeos-demo.com
LifeOS@123456
```

كلمة `admin` هنا لا تعني أنك تدير موظفين. معناها فقط أن حسابك الشخصي يملك صلاحية صيانة النظام وفتح صفحات الأمان والنسخ الاحتياطي وجودة البيانات.

## ماذا تستخدم يوميًا؟

ابدأ بأربع صفحات فقط:

1. مركز اليوم: متابعة الملخص العام.
2. الالتقاط السريع: تسجيل أي فكرة أو مهمة أو ملاحظة بسرعة.
3. مخطط اليوم: أهم 3 نتائج وتركيز اليوم.
4. المال والمصروفات: الدخل، المصروفات، الحسابات، والميزانية.

بعد أن تعتاد على النظام أضف:

- المهام والأهداف
- الصحة
- اليوميات
- المراجعات الأسبوعية

## ماذا تتجاهل في البداية؟

لا تحتاج هذه الأجزاء في أول استخدام:

- الأتمتة
- التنبيهات
- الاستيراد والتصدير
- القوالب
- سجل التدقيق
- إدارة الوحدات

هذه صفحات صيانة أو توسع، وليست ضرورية للتسجيل اليومي.

## ترتيب التشغيل من الصفر

شغل ملفات قاعدة البيانات بهذا الترتيب:

```text
database/schema.sql
database/mega-extension.sql
database/create-demo-auth-users.sql
database/first-admin.sql
```

اختياريًا، لو تريد بيانات تجربة:

```text
database/realistic-demo-seed.sql
```

## روتين يومي بسيط

صباحًا:

- افتح مخطط اليوم.
- اكتب أهم 3 نتائج.
- راجع المهام المفتوحة.

أثناء اليوم:

- سجل أي شيء سريع في الالتقاط السريع.
- سجل المصروف فور حدوثه.
- سجل المياه/الوجبة فقط إذا كان هذا هدفك الحالي.

مساءً:

- افتح مركز اليوم.
- أغلق أو حدّث المهام.
- اكتب سطرًا واحدًا في اليوميات أو المراجعة.

## روتين أسبوعي

مرة كل أسبوع:

- افتح المراجعات.
- اكتب الإنجازات والمشاكل والدروس.
- راجع صندوق المراجعة.
- حوّل العناصر المهمة إلى مهام أو أهداف.

## قاعدة مهمة

لا تحاول استخدام كل الوحدات مرة واحدة. النظام قوي لأنه واسع، لكنه يصبح مفيدًا فقط عندما تبدأ بروتين صغير ومتكرر.
