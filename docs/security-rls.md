# الأمان و Row Level Security

## المبادئ

- كل مستخدم يرى بياناته فقط.
- الأدمن يستطيع رؤية بيانات النظام حسب السياسات.
- تغيير الدور إلى أدمن يجب أن يتم بحذر.
- لا تستخدم Service Role Key داخل الواجهة الأمامية أبدًا.

## ما الذي تم تجهيزه؟

- RLS للجداول الأساسية.
- RLS لجداول Mega.
- RLS لجداول الأدمن.
- دالة `public.is_admin()`.
- منع تغيير الدور من مستخدم عادي.
- سياسات Storage Bucket باسم `lifeos`.

## ما الذي يحتاج إضافة مستقبلًا؟

- سجل تلقائي لكل عملية Insert/Update/Delete.
- تنبيهات عند فشل تسجيل الدخول المتكرر.
- تصدير مشفر للنسخ الاحتياطي.
- صلاحيات أكثر تفصيلًا من مجرد admin/user.
